The cookbook demonstrates some common use case scenarios of the APIs. Take these examples for a spin yourself for inspiration getting started with PreGod.

Here we refer to a `Note` as a list of actions or events generated by the instance, the feed of an instance therefore naturally contains multiple `Note`.

## Get Notes by an Instance

Supposedly that DIYgod `0xc8b960d09c0078c18dcbe7eb9ab9d816bcca8944` (`address`, you can also use an ENS name) wants to check his own feed related to his collectibles (`tag`) on Ethereum (`network`). To achieve that, we can simply query the `GET` endpoint by constructing the query following the structure shown below:

<!-- markdownlint-disable MD034 -->

https://pregod.rss3.dev/v1.1.0/notes/{**address**}?tag={**collectible**}&network={**ethereum**}

<!-- markdownlint-enable MD034 -->

:::info
There are many filtering options available to further customize the result, please refer to [Data and Filtering](../datasource).
:::

<Tabs>
  <TabItem value="noteGetReq" label="Request">

And send the constructed query. For the demonstration purpose, we set limit the result to just 3 with `limit=1`. Here we use curl:

```bash
curl -X 'GET' \
  'https://pregod.rss3.dev/v1.1.0/notes/DIYgod.eth?tag=collectible&network=ethereum&limit=3' \
  -H 'accept: application/json'
```

</TabItem>

## Get Notes by a List of Instances

If now DIYgod wants to check the recent `collectible` `mint` on `polygon` network of his friends Songkeys and Henry, he can use PreGod's `POST` endpoint that returns a list of `Note` for multiple instances.

<Tabs>
  <TabItem value="notePostReq" label="Request">

At the moment only 0x addresses are supported in `POST`. A request body can be formed in this way:

```json
{
    "address": [
        "0x08d66b34054a174841e2361bd4746ff9f4905cc2",
        "0x827431510a5d249ce4fdb7f00c83a3353f471848"
    ],
    "tag": ["collectible"],
    "type": ["mint"],
    "network": ["polygon"],
    "limit": 3
}
```

We use curl to send the request:

```bash
curl -X 'POST' \
  'https://pregod.rss3.dev/v1.1.0/notes' \
  -H 'accept: application/json' \
  -H 'Content-Type: application/json' \
  -d '{
    "address": [
        "0x08d66b34054a174841e2361bd4746ff9f4905cc2",
        "0x827431510a5d249ce4fdb7f00c83a3353f471848"
    ],
    "tag": ["collectible"],
    "type": ["mint"],
    "network": ["polygon"],
    "limit": 3
}'
```

</TabItem>

## Get Platforms Supported by PreGod

<Tabs>
  <TabItem value="platformGetReq" label="Request">

As PreGod supports over 100 different platforms, you can check what platforms are supported by each `tag` by querying the `/platforms` endpoint. For example, PreGod is able to index Nodes on Ethereum on the following exchanges (both DEX and CEX):

````bash

```bash
curl -L -X GET 'https://pregod.rss3.dev/v1.1.0/platforms/exchange?network=ethereum' \
-H 'Accept: application/json'
````

</TabItem>

  <TabItem value="profileGetRes" label="Response">
